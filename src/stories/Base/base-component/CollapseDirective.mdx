# Collapse Directive — `v-collapse`

> **Deprecated.** `v-collapse` no longer has any effect on `MclCollapseA`, `MclCollapseB`, or the base `Collapse` component. Those components now manage state internally via the `useCollapseState` composable. This directive will be removed in a future release.

---

## Migration

### Accordion groups

Replace `v-collapse` + `accordion` prop with `AccordionGroup` wrapper:

```vue
<!-- Before (deprecated) -->
<button v-collapse:item-1>Item 1</button>
<Collapse id="item-1" accordion="my-group">...</Collapse>

<button v-collapse:item-2>Item 2</button>
<Collapse id="item-2" accordion="my-group">...</Collapse>
```

```vue
<!-- After -->
<script setup lang="ts">
import { ref } from 'vue'
import { AccordionGroup, Collapse } from '@bobbykim/manguito-theme'

const panel1 = ref<InstanceType<typeof Collapse>>()
const panel2 = ref<InstanceType<typeof Collapse>>()
</script>

<template>
  <AccordionGroup>
    <div>
      <button @click="panel1?.toggle()">Item 1</button>
      <Collapse ref="panel1" id="item-1">...</Collapse>
    </div>
    <div>
      <button @click="panel2?.toggle()">Item 2</button>
      <Collapse ref="panel2" id="item-2">...</Collapse>
    </div>
  </AccordionGroup>
</template>
```

See the [Collapse component docs](?path=/docs/base-base-component-collapse-component--collapse) for full standalone and `v-for` accordion patterns.
