# Sidebar

Included in `@bobbykim/manguito-theme`

A slide-in panel that can be anchored to the left or right edge of the viewport. Includes focus trap, scroll lock, backdrop, and animated open/close transition. ARIA attributes (`role="dialog"`, `aria-modal`, `aria-labelledby`) are applied automatically.

```ts
import { Sidebar } from '@bobbykim/manguito-theme'
```

---

## Usage

### Open via component ref

The recommended way to trigger a sidebar is to hold a template ref and call `open()`.

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Sidebar } from '@bobbykim/manguito-theme'

const sidebarRef = ref<InstanceType<typeof Sidebar>>()
</script>

<template>
  <button @click="sidebarRef?.open()">Open Sidebar</button>

  <Sidebar ref="sidebarRef" id="my-sidebar" title="Navigation">
    <template #body>
      <nav>...</nav>
    </template>
  </Sidebar>
</template>
```

### Open via `:visible` prop

Bind a reactive ref to `:visible` for state-driven control.

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Sidebar } from '@bobbykim/manguito-theme'

const isOpen = ref(false)
</script>

<template>
  <button @click="isOpen = true">Open Sidebar</button>

  <Sidebar :visible="isOpen" @close="isOpen = false" id="my-sidebar">
    <template #body="{ close }">
      <p>Sidebar content.</p>
      <button @click="close">Close</button>
    </template>
  </Sidebar>
</template>
```

### Custom header, body, and footer

Use named slots to fully customise the sidebar layout.

```vue
<template>
  <Sidebar
    ref="sidebarRef"
    id="my-sidebar"
    placement="right"
    class-name="bg-light-1 p-sm"
    :width="360"
  >
    <template #header="{ close }">
      <div class="flex justify-end mb-4">
        <button @click="close" aria-label="Close">✕</button>
      </div>
    </template>
    <template #body="{ status }">
      <p>Open: {{ status }}</p>
      <nav>...</nav>
    </template>
    <template #footer>
      <div class="border-t pt-4">Footer content</div>
    </template>
  </Sidebar>
</template>
```

### Default header

Pass `title` to use the built-in header instead of the `#header` slot.

```vue
<Sidebar ref="sidebarRef" id="my-sidebar" title="Menu" color="primary">
  <template #body>...</template>
</Sidebar>
```

## API

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `id` | `string` | — | HTML `id` on the sidebar root. Used for ARIA `aria-labelledby` wiring. |
| `title` | `string` | — | Text for the built-in default header. Omit when using `#header` slot. |
| `titleColor` | `ColorPalette` | `'dark-3'` | Color of the default header title text. |
| `visible` | `boolean` | `false` | Initial open state. Reactive — changing it opens or closes the sidebar. |
| `color` | `ColorPalette` | `'light-1'` | Background color of the sidebar panel. |
| `class-name` | `string \| string[]` | `''` | Additional CSS class(es) on the sidebar panel. |
| `placement` | `'left' \| 'right'` | `'left'` | Which edge of the viewport the sidebar slides in from. |
| `width` | `number` | `300` | Width of the sidebar panel in pixels. |
| `no-backdrop` | `boolean` | `false` | When `true`, renders without the semi-transparent backdrop. |
| `no-header` | `boolean` | `false` | When `true`, suppresses the header area entirely. |
| `backdrop-color` | `ColorPalette` | `'dark-3'` | Color of the backdrop overlay. |

### Slots

All three slots receive `{ close: () => void, status: boolean }` as slot props.

| Slot | Slot props | Description |
|---|---|---|
| `#header` | `{ close, status }` | Custom header. Omit to use the default header (requires `title` prop) or `no-header`. |
| `#body` | `{ close, status }` | Main content area. |
| `#footer` | `{ close, status }` | Footer area below the body. |

### Exposed Methods

| Method | Signature | Description |
|---|---|---|
| `toggle` | `() => void` | Toggle open/closed. |
| `open` | `() => void` | Open the sidebar. |
| `close` | `() => void` | Close the sidebar. |

### Events

| Event | Payload | Description |
|---|---|---|
| `open` | `(visible: boolean)` | Emitted when the sidebar opens. |
| `close` | `(visible: boolean)` | Emitted when the sidebar closes. |
